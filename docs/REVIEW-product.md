# 시스템 기획 검토 체크리스트 — 회사 A의 여정 기준

> 2026-07-16, SaaS(멀티테넌트) 전환 직후 시스템 기획을 전체 검토한 결과.
> 회사 A가 가입해서 캠페인 성과를 보기까지의 여정을 따라가며, 각 단계에서
> **서 있는 것**과 **비어 있는 것**을 나눴다. 완료한 항목은 `[x]` + 근거를 남긴다.
> 우선순위: 🔴 실무 완성도(작음) · 🟡 있으면 좋음 · 🔵 로드맵 합류(큼).

## 1. 가입 → 워크스페이스 (회사 등록)

**서 있는 것**: 가입 = 워크스페이스 + 첫 계정 ADMIN, 역할 2단계(관리자/운영자),
행 단위 데이터 격리(+by-id 404), 역할 변경 즉시 반영(JWT에 역할 미탑재 설계).

- [ ] 🟡 **초대 링크 플로우** — 지금은 관리자가 초기 비밀번호를 만들어 전달.
      표준은 초대 메일 → 본인이 비밀번호 설정. 자기 발송 인프라로 초대 메일을 보내는 그림이 자연스럽다.
- [ ] 🟡 비밀번호 재설정/변경
- [ ] 🟡 멤버 비활성화/삭제 — 퇴사자 계정이 영구 유효한 상태
- [ ] 🟡 가입 이메일 소유 검증
- [ ] 🟡 토큰 갱신(refresh) — 현재 2시간 만료 후 재로그인
- [ ] 🟡 **로그인 시도 제한** — 현재 브루트포스 방어 없음(무제한 시도 가능).
      계정/IP 기준 백오프 또는 일시 잠금. BCrypt 비용이 유일한 방어인 상태.
- [ ] 🟡 비밀번호 정책 — 현재 길이 최소치뿐. 최소 강도 기준 + 유출 비밀번호 목록 대조 검토.
- [ ] 🟡 로그인 이력/알림 — 새 기기·이상 위치 로그인 기록(감사 축과 합류).

## 2. 수신자 확보 → 리스트

**서 있는 것**: "멤버십은 분류, 구독 의사는 별도 기록" 원칙(전역 억제 + 리스트
옵트아웃 2계층), CSV 재가져오기에도 해지 유지 — 이 시스템에서 가장 잘 선 기획.

- [x] 🔴 **수신 동의 기록** — *2026-07-16 완료.* V17 `consent_source`/`consented_at`,
      경로별 자동 스탬프(MANUAL/CSV_IMPORT), 레거시는 null="기록 없음"으로 정직 표기.
      드로어에 노출. (정통망법류 대응의 최소 단위)
- [x] 🔴 **수신자 화면 페이지네이션 + N+1 제거** — *2026-07-16 완료.*
      `GET /api/contacts/page`: 서버 필터(검색·리스트·구독·참여도) + 배치 3쿼리 사전 병합.
      수신자당 3콜 → 페이지당 1콜(13명 기준 41콜→3콜 실측). 페이저 UI 추가.

## 3. 콘텐츠 (템플릿)

- [x] 🔴 **빌트인 템플릿 copy-on-write** — *2026-07-16 완료.* 빌트인 update는 409(읽기 전용),
      `POST /api/templates/{id}/copy`로 내 워크스페이스에 복제. 카드 클릭이 곧 복사-후-편집.
- [ ] 🟡 업로드 이미지 워크스페이스 격리 (현재 공용 디렉터리)

## 4. 캠페인 작성 → 발송

**서 있는 것**: 임시저장→불러오기→예약(+취소)→기간(수집 컷오프)→A/B 승자→참여도
세그먼트 체인이 일관된 의미론(팬아웃 시점 평가)으로 이어짐.

- [x] 🔴 **테스트 발송("내게 먼저 보내기")** — *2026-07-16 완료.* `POST /api/campaigns/test-send`:
      샘플 변수 렌더 + "[테스트]" 접두, 파이프라인 밖(캠페인/메시지/지표 무기록). 폼에서 A/B 변형 선택 발송.
- [x] 🔴 **발송 전 최종 확인 단계** — *2026-07-16 완료.* 발송 등록이 요약 모달(캠페인/대상+세그먼트
      예상 인원/시점/기간/A/B)을 거친 뒤에만 큐 등록.
- [ ] 🟡 발송 중 일시정지/중단 (현재 취소는 릴리스 전 예약만)
- [ ] 🔵 **발신 도메인 검증(SPF/DKIM)** — 플랫폼 SES에서 고객 발신 도메인으로 보내기
      위한 관문(ESP 표준 플로우). 관리 페이지 발송 설정의 다음 칸.

## 5. 발송 파이프라인

**서 있는 것**: O(1) 생성, 비동기 팬아웃, 원자적 claim(20.5→157.7 msg/s 측정),
DLQ, 상태/이벤트 분리 — 가장 깊게 판 축.

- [x] 🔵 **테넌트별 발송 rate limit** — *2026-07-20 완료.* 워크스페이스 키 토큰버킷
      (V19, Postgres 원자적 UPDATE) + TTL 파킹 큐. 관리 페이지에서 발송 속도 설정.
      실측: A사 60건이 rate=3으로 도는 중 B사 10건 0.3초 완료.
- [ ] 🔵 soft bounce 재시도 정책 (ROADMAP Tier 3)

## 6. 추적 → 분석

**서 있는 것**: 이벤트 스트림 → 프로젝션 → distinct 집계, 기간 컷오프, 링크 랭킹, 히트맵.

- [ ] 🟡 봇/프록시 오픈 필터 (Gmail·Apple 프리페치로 오픈율 과대 — 업계 공통)

경위는 [ROADMAP 설계 노트](ROADMAP-scale.md)) + 미터링 + 테넌트 rate limit + 워크스페이스/역할.

      사용량 과금 모델의 남은 마지막 조각.

**결제 기능 보완 체크 (2026-07-23 추가 — 플랜/쿼터의 하위 과제):**

- [x] 🔵 **요금표 정의** — *2026-07-24 정책 문서 완성:* [BILLING-policy.md](BILLING-policy.md)

- [ ] 🔵 **청구서(인보이스) 생성·이력** — 월별 발송량 스냅샷 고정(정산 후 재집계로
      숫자가 변하면 분쟁) + 청구 내역 화면. 미터링이 "조회 시점 계산"인 현재 구조에서
      **월 마감 시점의 스냅샷 테이블**이 선행 과제.
- [ ] 🔵 **한도 초과 정책** — 월 쿼터 도달 시 발송 차단인지 초과 과금인지, 유예(grace)
      구간은 얼마인지. 차단이면 파이프라인의 어느 지점에서 막을지(생성 시 vs 디스패치 시).
