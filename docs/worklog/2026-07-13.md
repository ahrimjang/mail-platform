# 작업 로그 — 2026-07-13

A/B 테스트 기능 도입 — 분할 발송·변형별 지표에서 시작해 B안 템플릿, 새 캠페인 화면 개편, 승자 자동발송 플로우 구현까지. (자정 넘겨 이어진 작업은 07-14 로그에.)

## 오늘 한 일 (요약)
1. **A/B 분할 테스트(기본)** — 캠페인에 B안 제목/본문 + 분배 비율, 수신자 해시 기반 결정적 배정, dispatch가 B 메시지에 B안 렌더, 상세 API에 변형별 지표(`variants[]`). V8.
2. **엔티티 컬럼명 함정 수정** — `abBodyB`를 Hibernate가 `ab_bodyb`로 해석(트레일링 대문자) → `@Column(name=...)` 명시로 해결.
3. **B안 템플릿 지원** — `abTemplateId`로 B안도 템플릿 스냅샷(메인 `templateId`와 동일 패턴).
4. **새 캠페인 화면 1차 개편** — 컨테이너 560→880px, A/B 사용 시 A안/B안 좌우 2열 카드(각각 직접/템플릿), 템플릿 선택 시 미리보기, A/B 섹션을 내용 위로.
5. **승자 자동발송 플로우 구현** — 테스트 그룹 N%만 A/B 발송, 나머지는 보류(variant NULL·미발행), 평가 대기 후 오픈율/클릭율로 승자 판정 → 보류분 자동 발송. V9.
6. 시연 착수 — 분할·지표 E2E(수신자 12명, MailHog 분배 일치, 트래킹→변형별 오픈/클릭 반영) 확인.

## 상세

### A/B 분할 + 변형별 지표 (V8)
- 배정: `AbVariantAssigner` — 이메일 해시 버킷으로 결정적(같은 수신자는 항상 같은 변형 → 재시도·재확장에 안전). 인라인·팬아웃·예약 세 경로 동일 적용.
- 렌더: dispatch가 `variant=="B"`면 B안 제목/본문으로 치환(B 본문 없으면 제목만 테스트).
- 지표: `mail_messages`를 variant로 group by + `email_events`↔메시지 조인 distinct — `variants[] {variant,total,sent,opened,clicked}`.

### 승자 자동발송 (V9)
- 홀드아웃: 테스트 그룹 밖 수신자는 **variant NULL·PENDING·미발행**으로 생성(예약 발송의 지연 릴리스 패턴 재사용).
- 판정: 워커 스케줄러(30초)가 `ab_evaluate_at` 도래 캠페인의 변형별 참여율 비교 → **원자적 조건부 UPDATE claim**(`ab_winner IS NULL`)으로 정확히 1회 결정(동점→A 폴백) → 보류분 발행.
- 행 재작성 없음: dispatch가 "variant NULL + 승자 확정"이면 승자 콘텐츠로 렌더 → **비교 지표는 자동으로 테스트 그룹만** 집계됨.
- 완료 게이트 무수정: 보류 PENDING이 캠페인을 SENDING에 붙잡아 조기 COMPLETED 없음.

### UI (새 캠페인)
- 테스트 그룹 비율 슬라이더(10~50%) + 평가 기준(오픈율/클릭율) + 대기 시간 선택. 어색했던 A:B 숫자 입력 제거(내부 50:50 고정).
