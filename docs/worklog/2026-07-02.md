## 설계 결정
- 캠페인은 템플릿 내용을 **생성 시 스냅샷**(발송 중 템플릿 수정 영향 없음), `{{변수}}`는 **발송 시점에** 연락처 `toVariables()`로 렌더. raw 수신자는 `{email}`만.
- Contact에 **status 두지 않음** — suppression 목록이 do-not-send 단일 진실원천.
- 트랜잭셔널 = 요청 변수로 즉시 렌더 후 **단건 캠페인** 생성 → 큐·추적·억제·지표 전부 재사용.
- 캠페인 생성: `templateId`/`listId`(옵션) — 직접 입력(subject/body/recipients)과 공존.
- 리스트 fan-out 시 `MailMessage.contactId` 저장 → dispatchOne이 연락처를 로드해 개인화.
