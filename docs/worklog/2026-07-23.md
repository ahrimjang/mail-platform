## 오늘 한 일 (요약)

### 1. 과금 모델 전환 — BYO를 접다 (V20, `731eb9b`)
V16 때 "공유 SaaS + BYO 커넥터"로 정했던 과금 모델을 번복하고, **발송(SMTP/SES)·저장(S3)
인프라는 플랫폼이 직접 소유·연동, 고객에게는 월 발송량 기준 과금**(SendGrid류 표준 ESP
모델)으로 전환했다. 번복 근거 3가지를 [ROADMAP 설계 노트](../ROADMAP-scale.md)에 원래
결정 바로 옆에 기록:

1. **온보딩 마찰** — BYO는 고객마다 자기 AWS 계정·샌드박스 해제·자격증명 등록을 요구.
   "가입하고 바로 보낸다"가 안 되는 첫 경험은 이메일 SaaS에 치명적.
2. **BYO의 명분 소멸** — "미터링·정산 없이 비용을 고객에게 직접 청구"가 원래 명분이었는데,
   그 사이 월 발송량 미터링과 테넌트별 rate limit이 구현돼 사용량 과금의 전제가 완성됨.
3. **발신 평판 통제** — IP/도메인 웜업·바운스 관리는 인프라를 쥔 쪽만 할 수 있다.

구현: V20으로 `workspaces`의 프로바이더 선택 컬럼 제거(선택만 저장·실연동 없던 상태라
데이터 손실 없음), 도메인~DTO~`types.ts`에서 BYO 흔적 정리, 관리 페이지의 BYO 카드 →
**"사용량 · 요금" 카드**(월 발송량 = 청구 기준으로 승격) + **"발송 설정" 카드**(속도 제한
독립)로 재구성. 문서(README·CLAUDE.md·logic 10/11·REVIEW 7절) 서사 전환. 검증: V20 부팅
적용 확인, 브라우저에서 카드 구성·"BYO" 텍스트 0건·새 요청 형태 저장 왕복 확인.

### 2. 산출물 1번 — README 아키텍처 다이어그램 + 숫자표 (`d98691a`)
- **Mermaid 다이어그램**: 인바운드 3종(콘솔/수신자 추적/SNS 웹훅) → api → 상태·스트림
  (Postgres/RabbitMQ/Kafka) → 워커 N대 → SMTP 릴레이, 관측 축은 점선. 화살표 라벨에
  설계 포인트("캠페인 1행+잡 1개 O(1)", "원자적 claim", "동시성 8×N")를 실음.
- **before/after 숫자표**: 생성 12.1초→40~66ms(~240×, O(N)→O(1)) · 드레인 20.5→157.7
  msg/s(~7.7×) · 동시 소비자 12에서 이중발송 0 · noisy neighbor(rate=3 준수 중 무제한
  테넌트 0.3초). 전부 [loadtest/RESULTS.md](../../loadtest/RESULTS.md) 실측값.
- "측정이 추측을 이긴 순간들"(SMTP 병목 가설 기각, 증설보다 동시성, 실측이 잡은 NPE)을
  표 아래 한 문단으로.
