# 작업 로그 — 2026-07-20

확장성 Tier 2의 남은 두 축을 닫았다: **테넌트별 발송 속도 제한(토큰버킷)** 과
**메트릭 대시보드(Micrometer→Prometheus→Grafana)**. 커밋 2건 푸시.

## 오늘 한 일 (요약)

### 1. 테넌트별 발송 속도 제한 — Postgres 토큰버킷 (V19, `d23ee8c`)
멀티테넌시가 만든 noisy neighbor(A사 100만 건 뒤에 B사 10건이 줄 서는 문제)와
BYO 프로바이더 한도 대응을 한 장치로. 상세: [logic/11-send-throttling.md](../logic/11-send-throttling.md).

- **설정은 워크스페이스가 스스로**: 관리 페이지 BYO 카드에 "발송 속도 제한(건/초)"
  (`workspaces.send_rate_per_sec`, 비우면 무제한) — "연결한 SES의 초당 한도를 적어 두면
  그 속도로 나눠 발송"이라는 BYO 서사와 결합.
- **버킷은 Postgres 원자적 조건부 UPDATE 한 문장**(리필+차감, `workspace_send_buckets`).
  ROADMAP 초안은 Redis였지만, 이 저장소의 동시성 불변식 계보(발송/팬아웃 claim, 예약
  릴리스, A/B 승자)를 따르면 새 인프라 없이 워커 N대 정합성이 보장된다. 병목이 오면
  `SendRateLimiter` 포트 뒤에 Redis를 꽂는 것으로 로드맵에 기록.
- **거절된 메시지는 TTL 1초 파킹 큐로**(`mail.send.throttled`, 소비자 없음 → 만료 시
  발송 큐 꼬리로 dead-letter): nack 재큐의 hot loop도, sleep의 소비자 슬롯 잠식도 없이
  한도 초과 테넌트만 비켜서고 다른 테넌트는 그대로 흐른다.
- **토큰 확인은 claim보다 먼저** — claim 후 거절이면 메시지가 SENDING에 갇혀 stale
  재클레임 창(2분)을 기다려야 한다. 끝난 메시지의 재배달은 토큰을 쓰지 않는 가드도.
- **실측**: rate=5 → 40건 7.7초(실효 5.2 msg/s) · rate=3 → 60건 19.8초(정확히 3.0) ·
  그 백로그가 도는 중에 무제한 테넌트 10건은 **0.3초**(noisy neighbor 해소 실증).
- **실측이 잡은 결함**: 무제한 워크스페이스(rate NULL)에서 `stream().findFirst()`가
  null 요소에 NPE → 메시지가 전부 DLQ행. rate 있는 경로만 커버한 단위 테스트는 통과했던
  버그 — E2E 단계에서 발견·수정하고 DLQ 10건도 재발행으로 복구.

### 2. 메트릭 대시보드 — Micrometer → Prometheus → Grafana (`858905b`)
부하테스트가 "한 번 재는" 도구였다면 이건 상시 계기판. 상세: [logic/12-metrics-grafana.md](../logic/12-metrics-grafana.md).

- **계측은 어댑터에만**(코어는 Micrometer를 모름): SMTP 발송 타이머(`mail.smtp.send`,
  outcome 라벨 — rate()가 곧 발송 처리량), 스로틀 판정 카운터(`mail.throttle.acquire`),
  발행 카운터(`mail.enqueue`). 전역 레지스트리 기록이라 actuator 없는 앱에선 no-op.
- **워커에 메트릭 전용 HTTP**(:8082, `/actuator/**`만 — 비즈니스 엔드포인트 없음),
  api는 :8080 `/actuator/prometheus` permitAll. RabbitMQ는 prometheus 플러그인(:15692),
  큐별 깊이는 detailed 엔드포인트를 별도 스크레이프 잡으로.
- **전부 파일 프로비저닝**: compose에 Prometheus(9090)+Grafana(3000, 익명 열람), 데이터
  소스·대시보드가 파일로 선언돼 `down -v` 후에도 같은 화면, 변경이 git diff로 리뷰됨.
- **대시보드 6패널**(RED·USE를 파이프라인 단계에 대응): 발송 처리량 / 큐 깊이 /
  테넌트 스로틀(거부·승인·재큐) / SMTP p50·p95 / API 요청 / JVM 힙.
- **검증**: 타깃 4개 UP, rate=4 캠페인 80건에서 토큰 거부 10.5/s ≈ 파킹 재큐 10.7/s
  (두 시리즈 일치 = 스로틀 동작이 그래프로 증명), 파킹 큐 깊이 18 출렁임 관측.
